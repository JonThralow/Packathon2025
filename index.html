Event Rules & Guidelines

Participants must adhere to the following rules to ensure a fair and productive hackathon:

  • 🛡️ Follow the code of conduct: Respect all participants, no harassment, and maintain a positive environment.
  • 👤 Participation: Participants must compete solo; no teams allowed.
  • 💻 Coding: All code must be written during the event. Pre-existing code is not permitted except for the provided VieroMind repo.
  • 📂 Git Repository: Create a separate Git repository named "[participant-name]-vieromind" for your submission. Make it public and share the link during demos.
  • 🛠️ Tools: Use any programming languages or frameworks, but ensure compatibility with the VieroMind base.
  • 📝 Submissions: Include a demo video (max 3 minutes) and README with setup instructions.
  • ⚖️ Judging: Based on innovation, impact, technical execution, and presentation.
